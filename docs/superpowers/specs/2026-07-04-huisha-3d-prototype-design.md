# 《回煞》3D 原型入口设计

日期：2026-07-04

## 目标

第一阶段不重写整部游戏，而是在现有 Phaser 2D 版本旁边新增一个可玩的 3D 原型入口。玩家可以从标题页进入 3D 体验，先探索正厅与走廊小段，感受第一人称/近第一人称的老宅空间、光照、交互提示和恐怖氛围。现有 2D 主流程必须保留，当前 `src` 维护方式和单文件 `index.html` 构建链路也必须继续可用。

## 交付范围

本阶段采用 Three.js 做 3D 原型层，不替换 Phaser 主流程。新增入口建议放在标题页，提供“开始游戏”和“3D 原型”两个选择；默认开始游戏仍走原来的 2D 序章，避免破坏当前可玩版本。3D 原型进入后加载一套独立场景：正厅、供桌、棺材、黑布相框、通往走廊的门洞，以及走廊尽头的氛围封锁。玩家能用键盘移动、鼠标或指针控制视角，靠近物件时显示调查提示，按交互键弹出复用现有 HTML 对话框或轻量浮层。

## 技术方案

新增 `src/3d/` 目录承载 3D 代码，尽量和现有 Phaser 场景解耦。建议模块划分为：

- `ThreePrototypeScene.js`：创建 renderer、camera、灯光、场景、动画循环和销毁逻辑。
- `ThreeHouseMap.js`：用数据描述房间尺寸、墙体、门洞和关键物件，第一阶段用程序化低模几何体。
- `ThreeControls.js`：处理 WASD、鼠标视角、指针锁定/拖拽和移动碰撞。
- `ThreeInteraction.js`：处理射线或距离式交互、提示和对话回调。
- `ThreePrototypeScene.test` 或验证脚本：至少检查模块导出、地图契约和入口构建不丢依赖。

Three.js 依赖优先使用本地文件或稳定 CDN 兜底。若使用外部 CDN，`index.html` 直开能力会受网络影响；因此更推荐把 Three.js 依赖落为本地静态文件或使用简单模块文件随项目发布。

## 交互与体验

3D 原型要服务“能玩一小段”，不是单纯看模型。玩家进入后站在正厅门口，手电光或锥形光照引导视线；供桌、棺材、黑布相框至少有调查反馈；走向侧门可以进入一段短走廊，走廊尽头以锁门/暗影对白收束，避免第一阶段做过大地图。退出方式必须明显，例如按 `Esc` 或点击“返回标题”，防止玩家被困在原型里。

## 风险与边界

第一阶段不做完整 3D 资产、不做全地图迁移、不重写剧情状态机、不接入复杂物理引擎。碰撞可以先用简单 AABB 或圆形半径约束。移动端先保证可打开和可退出，优先支持桌面键鼠体验。后续如果验证手感成立，再逐步迁移厨房、后院、追逐和记忆地图。

## 验证标准

代码完成后必须运行现有验证脚本：`node tools\verify_story_state.mjs`、`node tools\verify_maps.mjs`、`node tools\verify_start_route.mjs`。若改动 `src` 后同步单文件入口，必须运行 `node tools\build_standalone_entry.mjs` 和 `node tools\verify_standalone_entry.mjs`。3D 原型还需要补一个静态契约验证脚本，确认 3D 入口文件、核心模块和必要场景物件存在。最终用浏览器打开本地页面，确认 3D canvas 非空、能移动、能交互、能返回标题。

## Git 要求

本任务完成后必须自动执行 git 提交并推送 GitHub。若中间产生设计提交，也要保持 `main` 与 `origin/main` 对齐；实现提交完成后再次推送，确保老板不需要二次提醒。

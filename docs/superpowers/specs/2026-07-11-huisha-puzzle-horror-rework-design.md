# 《回煞》解谜、沉浸与恐怖机制重构设计

## 目标与已证实问题

本轮目标不是继续增加文档或延长跑图，而是让玩家在约 35–45 分钟内持续经历“观察环境、形成判断、实际操作、看到世界回应”的闭环。完整路线结束后，玩家既能复述“没有回家吃饭的人其实是自己”，也能记住至少三次由自己推理或行为造成的恐怖变化。

厨房关键道具问题已经在浏览器稳定复现。饭柜的交互优先级为 10，纸人优先级为 30、半径为 120；现有评分 `priority * 1000 + facing * 200 - distance` 让优先级绝对压过距离。玩家从所有可达方向靠近饭柜时，即使饭柜只有 18 像素远、纸人有 43–96 像素远，目标仍然固定落在纸人。饭柜还被通用柜子规则标成“躲藏”，与实际取得倒头饭的行为不符。

玩法审计同时确认：倒头饭、香、纸钱都是一次交互直接获得，火柴进入密室后也可直接取得；学校谜题把 1986、1987、1988 写在选项上，医院谜题把诊断、处方、未缴费单写在选项上。玩家不需要保存信息、提出假设或排除错误，只是在执行开发者清单。剧情虽已有共同母题，但主要因果仍由文档和回响直接告知，玩家没有通过操作亲手证明这些结论，因此读完容易遗忘。

## 设计取向

采用“纵向重构”，不引入长期随机巡逻怪、弹药、饥饿、有限背包或程序化密码。长期生存压力会与现有短流程和多结局探索冲突，也容易重新制造追逐坐牢。本轮保留 18 张地图、四个结局、4 秒跨图追逐和单文件入口，重做供品获取、两段记忆谜题、供桌仪式以及三类行为型恐怖机制。

谜题遵循四段闭环：先在世界中发现两到三条具体证据；再把证据转译成一个空间、因果或排除规则；随后在物件或证据板上操作；最后让房间、人物或剧情状态产生可见变化。谜面不得把答案直接写在选项标签中，错误反馈必须指出哪条假设冲突，但不直接公布完整答案。

本轮借鉴的不是具体题目，而是成熟作品的方法：`The Room` 用物件检查与隐藏机构让答案存在于可操作世界中，`Return of the Obra Dinn` 让玩家根据证据自行确认因果。实现仍使用《回煞》现有 2D 地图、程序化贴图和 DOM 浮层，不复制外部作品的谜题或视觉资产。

## 统一玩法主线：最后一顿饭

所有主要谜题共同回答一个问题：七月十四那顿饭，谁坐在哪里、谁阻止了谁、谁最终没有回来。

道具不再只是供桌所需的四个布尔值。倒头饭证明家里一直给缺席的孩子留位；火柴证明父亲用纸扎和封条把“家”固定成循环；香把母亲井边的等待引回屋内；纸钱代表父亲留下的债。学校谜题证明父亲改写了最后争吵的时间，医院谜题证明母亲的治疗不是自然中断。供桌最终要求玩家利用这些结论还原仪式，开棺再把所有操作翻译成“这场祭祀一直在招我自己回来”。

剧情推进不再主要依赖收集数量。每个大谜题完成后写入一个 `caseConclusions` 结论：`empty_seat`、`rewritten_night`、`treatment_blocked`、`sealed_house`、`unpaid_debt`。NarrativeDirector 根据结论组合播放短回响；原始文档继续提供证据，但不再承担独自解释整段剧情的职责。

## 交互选择修复

候选目标不再让远处高优先级物件绝对抢占近处目标。选择流程改为：先过滤超出各自半径的对象；再找出物理边缘距离最近值；只在“最近值 + 28 像素”的焦点带内比较剧情优先级和朝向。玩家几乎贴着一个物体时，该物体必定可选；距离接近时，关键剧情物才利用优先级胜出。正对物体获得朝向优势，背对且未贴身的物体不会抢焦点。

饭柜显式声明 `label: 封死的饭柜`、`verb: 检查`、剧情优先级和标记，不再继承柜子的“躲藏”。纸人、饭柜、水槽、灶台分别拥有不重叠的主要站位；浏览器回归必须从饭柜下方、左下和右下三个可达点确认提示和执行目标都为饭柜。

## 供品获取

### 倒头饭：厨房座次谜题

厨房是完整推理闭环的教学。水槽提供碗的使用痕迹，灶台灰痕提供桌子朝向和门、灶的位置，纸人只在被正视时停下并以手势指向明儿过去的座位。饭柜打开后显示三只日常碗与一只供碗，玩家需要把父亲、母亲、明儿三个名牌放到正确座位，再指出哪个空位该摆倒头饭。

线索不会直接出现“答案是东位”。父亲的酒碗、母亲的药渍碗和明儿的纸飞机刻痕负责身份映射；“父亲背门、母亲近灶、明儿挨着能逃的侧门”负责空间映射。完成后才获得倒头饭和 `empty_seat`。错误摆位会高亮一条发生冲突的已读证据，纸人推进一个锚点；第三次错误触发短暂熄灯，恢复后纸人站在饭柜旁，但谜题进度只清除冲突位置，不重置整张地图。

### 火柴：封宅证据链

密室内的火柴盒改为空盒，划火片被压在四张纸扎封条下。父亲笔记提供封条用途，墙上木偶影子提供四个方位。玩家需要按“门、窗、井、棺”与对应方位拆下封条；错误拆取会让已经打开的暗门缓慢合拢并传来一次门外脚步，但不会永久锁门。完成后取得划火片与火柴，同时写入 `sealed_house`，确认父亲不是简单做法事，而是在把一家人的执念封在宅中。

### 香：井边结绳

香不再直接放在树后。井绳上有三种结和三张受潮符纸，厨房纸人、地下室铁链与母亲病历分别给出“孩、门、病”的图形对应。玩家必须解开代表孩子的结，其他结会收紧井绳并触发井下呼吸声。正确后井绳带出香和母亲留下的一句短录音式回响；血红钥匙仍由井事件取得，现有追逐起点不改变。

### 纸钱：阁楼债主排除

阁楼悬着父、母、子、无名四束纸钱。破窗看到的房屋方向、父亲账页和全家福缺角共同证明真正被长期祭奠的是“没有尸体、没有牌位、照片缺一角”的孩子。玩家选择对应纸束后取得纸钱并写入 `unpaid_debt`。错误选择不会扣除道具，只会让楼下对应房间传来一声撞门，并在纸束背面留下新的排除提示。

## 记忆谜题重做

### 学校：证明被改写的雨夜

移除年份排序。玩家在夜校记忆中取得留堂钟、末班车票、被涂改的成绩单和父亲写给老师的说明。证据板提供三个待确认结论：“离校时间”“到家最早时间”“父亲声称的争吵时间”。玩家把证据放入结论槽并标出唯一不可能同时成立的说法。

正确推理应证明：李明不可能在父亲声称的时间已经到家，父亲后来改写了最后争吵的时间。完成后写入 `rewritten_night`。错误组合提示具体冲突，例如“这张车票只能证明发车时间，不能证明已经到家”，而不是只显示“顺序错误”。

### 医院：找出治疗中断点

移除“诊断 → 处方 → 未缴费单”的字面排序。玩家取得七日处方、药柜余量、两次病房记录与一张被撕去日期的缴费单。证据板显示七天药盒，玩家依据每日剂量和剩余药量判断实际停药日，再选择哪条记录能证明停药来自父亲阻断而非母亲拒绝。

完成后写入 `treatment_blocked`。错误日期会根据药量显示“余量对不上”，错误归因会指出证据只能证明现象还是能证明行为。谜题计算使用纯函数，不依赖 UI 文本硬编码答案。

## 供桌仪式

集齐四样供品后不自动完成。供桌出现四个位置：留位、引魂、偿债、照真。玩家必须根据已确认结论把饭、香、纸钱、火柴放入对应位置，并最后选择点燃顺序。正确操作依次改变遗像、饭碗和房间光线；遗像最后短暂变成少年李明。

若关键结论不足，供桌只显示缺少哪类理解，例如“还不知道这顿饭在等谁”，而不是直接报缺哪个地图。错误放置触发一次对应人物的短回响并退回该件供品，不消耗物品、不重跑获取流程。仪式完成才写入 `ritualSolved`，替代单纯的 `candlesLit` 作为大门与棺材的叙事门槛；旧状态通过归一化兼容。

## 恐怖机制

新增 `HauntingDirector`，只管理有规则、可读且有冷却的恐怖压力。它不在文档、对话、谜题浮层或场景切换期间启动，不改变 4 秒追逐生成规则，也不制造随机秒杀。

第一类是“背视移动”。厨房纸人和走廊黑影使用离散锚点；玩家朝向它们且视线无遮挡时停止，背对并移动一定距离后推进。锚点不会堵门或覆盖关键物件，达到最近点只触发熄灯、耳语或理智下降，然后退回，不进入抓捕循环。

第二类是“数脚步”。完成学校记忆、医院记忆和首次供桌摆放后各有一次作者设定的监听窗口。门外声音会跟随玩家最近几步计数，HUD 显示“它在听”。玩家静止约 2.5 秒则声音离开并留下剧情耳语；继续移动会让门口出现黑影、关掉局部灯光并降低理智，但不重置进度。每个节点最多触发一次。

第三类是“错误有后果”。厨房摆位、密室封条、井绳、阁楼纸束和供桌的错误操作分别改变对应房间，而不是统一弹出“错误”。变化包括纸人靠近、暗门合拢、井绳收紧、楼下撞门、遗像转头。恐怖反馈必须在 0.3–1.5 秒内发生，随后允许玩家继续思考。

全局随机事件降为低频环境层，不再承担核心惊吓；随机镜头抖动不得与谜题反馈叠加。所有相机震动保持短时、低强度，禁止重新加入持续模糊。

## 数据与模块边界

`PuzzleState.js` 提供纯函数：谜题前置判断、槽位分配、冲突检测、完成判定和错误提示键。`Puzzles.js` 只保存证据、槽位、允许关系、答案约束和成功结论，不把运行状态存进配置。

`StoryState.js` 新增并归一化 `puzzleProgress`、`caseConclusions`、`ritualSolved`、`hauntingSeen`。任何谜题完成先写结论，再由 NarrativeDirector 和 ObjectiveManager读取。旧的 `puzzles.school/hospital` 继续作为结局兼容字段，由新结论完成时同步写入。

`HauntingDirector.js` 负责事件资格、冷却、纸人锚点、监听窗口和清理；GameScene 只在生命周期中创建、更新和销毁它。MapManager 依据地图配置创建锚点和恐怖反馈目标，不在地图数据中存回调函数。

现有 DOM 谜题浮层扩展为“证据区 + 结论槽/空间槽 + 状态提示”。鼠标点击、键盘焦点、触摸操作使用同一数据接口；390×844 竖屏必须能滚动查看全部证据并始终看到关闭按钮。谜题打开时 `window.dialogActive` 或等价输入锁必须阻止玩家移动和恐怖事件更新。

## 目标与提示

HUD 不再连续列出地图清单。主线目标先显示当前问题，例如“找出饭桌上空着的是谁的位置”；当前区域提示指出可调查类别，例如“水槽、灶台和纸人的手势都留下了座次”；记忆行只显示已经由玩家证明的结论。

同一谜题连续失败两次后才开放一级提示，提示只重述一条已收集证据；连续失败四次开放二级提示，指出发生冲突的两个槽位；不提供一键完成。退出谜题保留已确认的非冲突分配，换房保留大谜题进度，开始新游戏完全清空。

## 错误处理与防坐牢边界

任何供品不得被永久消耗，错误答案不得锁死门、删除关键对象或开启无限追逐。恐怖反馈结束后必须恢复灯光、物理、输入和交互目标。纸人/黑影锚点生成前检查墙体、家具和门口安全区；找不到安全点则跳过视觉实体，只播放声音。

谜题配置缺失、证据 ID 不存在或旧存档字段异常时，浮层显示可关闭的错误信息并保留玩家状态。场景关闭时清除监听计时器、锚点实体、灯光 tween、DOM 回调和输入锁。厨房饭柜即使谜题配置异常，也必须能弹出明确错误而不是继续被纸人抢交互。

## 测试先行与验收

交互回归先用失败测试固定厨房问题：在三个可达采样点，饭柜比纸人更近时必须选中饭柜；两个目标距离处于同一焦点带时剧情优先级才可生效；饭柜提示必须为“检查：封死的饭柜”。浏览器随后按真实移动路线复验，不能只传纯函数坐标。

纯函数测试覆盖五套谜题配置：前置证据、合法分配、冲突说明、错误不污染完成状态、正确完成写入对应结论；供桌测试覆盖结论不足、错误退回、正确完成和旧状态归一化。HauntingDirector 测试覆盖对话/谜题期间不触发、背视才移动、监听静止成功、移动失败反馈、单节点只触发一次、场景销毁后无计时器。

自动化之外必须从新档完整游玩一次高真相路线，并对低、中出口做直达回归。完整游玩不使用开发者宝箱、状态注入或地图查询参数，记录总时长、每个谜题首次理解时间、错误次数、无目标徘徊超过 20 秒的位置、被错误物件抢交互的位置、恐怖事件是否打断阅读，以及任何剧情因果不成立之处。问题记录写入 `docs/playtests/2026-07-11-full-route.md`，能在本轮安全修正的必须继续修改后重跑相关段落。

最终验收包括全部 `tools/verify_*.mjs`、所有 `src/**/*.js` 语法检查、单文件重建、桌面与 390×844 浏览器回归。完成标准是：厨房关键道具可稳定操作；四件供品都至少经过两个有意义步骤；学校、医院和供桌不再是答案外露的排序题；剧情由至少五个玩家亲手证明的结论串联；三类恐怖机制有规则、有反馈且不会造成循环抓捕；完整路线无开发者辅助可通关。

# 《回煞》2D 稳定性与流程重构设计

## 背景与目标

当前 `main` 已同时包含 Phaser 2D 主线和独立 Three.js 3D 原型。玩家反馈集中在碰撞不稳定、结局条件混乱、流程像“坐牢”以及玩法缺少新意。现有六项验证脚本全部通过，但只证明静态契约存在，没有覆盖冲刺、移动端输入生命周期、追逐失败、静态物理体同步、谜题门槛和完整结局路径。

本轮目标是在保留现有 2D 地图、程序化美术和多结局骨架的前提下，完成一次可感知的 2D 版本收束：清退 3D 产品入口，修复已证实的稳定性问题，把学校、医院、照片走廊和追逐从单纯调查文本改成有目标、有反馈、有选择的短玩法，并让四个结局都能通过清晰条件正常到达。

## 范围拆分

本轮按三个连续工作流实施，但作为同一个 2D 版本交付：

1. 产品清理：移除 3D 运行时、入口和验证脚本，归档 3D 美术及历史设计资料。
2. 稳定性修复：解决碰撞、冲刺、追逐重试、帧率差异、移动端监听累积和场景氛围错误。
3. 流程与内容重构：加入目标提示、两个记忆谜题、全家福反制追逐以及明确的结局门槛。

不重写 Phaser 架构，不新增外部框架，不制作大规模新地图，不引入战斗系统，也不重写 Git 历史。

- `verify_standalone_entry.mjs` 对 3D 入口的要求

## 玩家流程与节奏

2D 主循环保留“雨夜车祸 → 进入老宅 → 收集供品与真相 → 井边钥匙与追逐 → 供桌开棺 → 结局选择”，但每一段都必须向玩家展示当前目的。

页面新增一行轻量目标提示，不做任务列表和地图导航。目标文案由统一进度状态推导，例如“进入老宅”“找齐供品：倒头饭、火柴、香、纸钱”“调查旧书房与药柜小间”“设法摆脱黑影”“回到供桌完成仪式”。玩家再次调查供桌时仍会看到缺少的具体物品。

对白保留打字机效果，但允许键盘和移动端点击先补全、再继续。长对白继续自动分页，不再用额外等待拖慢流程。章节失败不清空整局状态：追逐失败从当前地图的安全出生点重试，保留已获得物品和线索，同时重置追逐者位置和临时躲藏状态。

## 地图用途契约

`Maps.js` 中每张地图都必须声明非空的 `purpose` 和 `rewards`。`purpose` 说明玩家为什么要进入，`rewards` 只能来自推进物品、真相线索、谜题、追逐解法、安全点或结局选择。没有独特产出、只增加步行距离的房间不得保留为独立地图。目标系统在玩家首次进入地图时显示该地图当前可完成的目的，完成后不再反复提示。

现有 18 张地图的职责固定如下：

| 地图 | 必须提供的用途与产出 |
| --- | --- |
| `room_prologue` | 教会移动与调查，建立车祸循环，明确指向老宅 |
| `room_entrance` | 老宅入口与阶段性离开选择，承载“破茧 / 回煞”出口 |
| `room_main` | 供品仪式、棺材真相、全家福反制追逐的高潮空间 |
| `room_kitchen` | 取得倒头饭，并触发纸人第一次位置变化 |
| `room_corridor` | 房间枢纽，收集四张照片并形成残缺全家福 |
| `room_bathroom` | 提供保险箱密码的替代提示，并揭示母亲自伤未遂的病情线索 |
| `room_bedroom_parents` | 家规、保险箱、地下室钥匙和密室入口 |
| `room_secret` | 取得火柴，并补充父亲制作纸人与封宅的证据 |
| `room_study` | 收集父亲控制线索并进入学校记忆 |
| `room_medicine` | 收集治疗中断线索并进入医院记忆 |
| `room_bedroom_me` | 母亲日记、纸飞机死亡线索和追逐躲藏点 |
| `room_backyard` | 取得香与血红钥匙，启动短追逐 |
| `room_basement` | 找到全家福缺角和锁门铁链证据，完成高真相路线所需的全家福 |
| `room_attic` | 取得纸钱，并提供俯视老宅的方向提示 |
| `memory_school` | 完成试卷顺序谜题并确认父亲控制主题 |
| `memory_hospital` | 完成治疗证据链谜题并确认母亲病情主题 |
| `memory_crash` | 调查死亡现场并进行高真相最终选择 |
| `room_memory` | 对“回到家”作最后确认并承载“永远在一起”结局 |

地下室保留 `room_basement` ID，但重新设计为“地下禁闭室”。地图从当前 30×30 收紧到 18×14，使用一条能看见返程方向的主通道，不保留假岔路和大面积空地。玩家进入后目标固定为“找到全家福缺角，再从原门返回”；入口、缺角和出口使用连续光源引导。房内保留锁门铁链、儿童抓痕和照片缺角三个调查点，其中照片缺角是完整真相的必要产出。

走廊四张照片调查完后只写入 `photoSetCollected = true` 并获得【残缺全家福】；地下室取得缺角后写入 `familyPhotoCornerFound = true`。两项同时满足时才生成【完整全家福】并写入 `familyPhotoAssembled = true`。这样地下室服务高真相路线，但低真相和家庭真相玩家仍可选择不进入，不会被强制多跑一张地图。

## 记忆谜题

学校和医院记忆不再允许直接调查终点返回。两张地图共用一个移动端友好的轻量谜题浮层，以点选顺序代替拖拽。

学校谜题要求玩家先调查黑板评语和成绩单，再把标有 1986、1987、1988 的三张试卷按年份升序排好。正确顺序揭示“最后一次争吵”，写入 `puzzles.school = true`，然后开放返回物件；错误顺序只清空当前选择并给出文本提示，不扣状态、不重开地图。

医院谜题要求玩家先调查诊室窗口和病房门牌，再按“诊断 → 处方 → 未缴费单”的证据链拼出治疗被阻断的过程。完成后写入 `puzzles.hospital = true`，开放返回物件。两个谜题都必须能用鼠标、键盘和移动端按钮完成。

记忆返回老宅后，不只播放一句对白：学校完成后走廊照片出现新的红笔痕迹；医院完成后药柜与走廊环境增加消毒水色调和新调查文本。变化保持轻量，使用现有程序化贴图和 tint，不新增外部美术依赖。

## 全家福与追逐

走廊四张照片继续沿用现有调查逻辑，但看完后不再只设置无消费的 `corridorSolved`。系统会生成【残缺全家福】并引导玩家前往地下室寻找缺角；取得缺角后才写入 `familyPhotoAssembled = true` 并刷新目标。

井边获得血红钥匙后触发父亲黑影追逐。追逐改成短遭遇，而不是永久跨房间追逐。玩家有两种脱困方式：在床或柜子中连续保持隐藏 6 秒，或带着全家福返回正厅，在供桌附近举起照片使黑影停下。全家福路线更主动，并为完整真相提供必要条件；躲藏路线仍允许普通玩家继续流程。

追逐成功后写入 `chasePhase = 'escaped'`，停止追逐、心跳和周期性噪音。追逐失败写入临时重试状态，在当前章节安全点恢复，不回序章，不保留旧追逐者实例，也不重复注册输入监听器。

## 正史统一

剧情正史统一为以下版本：

- 李明十年前因父亲控制和争吵离家，在雨夜车祸中死亡。
- 母亲长期未得到治疗，曾有自伤未遂，后来在失去儿子后投井。
- 父亲封锁老宅并守着空棺，最终死于宅中，但控制和悔恨形成黑影。
- 游戏中的“今晚车祸”是十周年忌夜被老宅执念拉回的记忆循环，李明误以为事故刚刚发生。
- “回煞”描述执念把亡者拖回旧宅，不再把故事解释成字面上的头七。

README、剧情母稿和游戏对白最终都以此版本统一。纸人作为老宅执念的引路与误导媒介，父亲黑影承担追逐角色，避免继续把同一张纸人贴图同时解释成奶奶、母亲和父亲。

## 四结局路由

结局判定集中为纯函数，不再由出口、棺材和记忆物件各写一套条件。

“破茧”是低真相离开。玩家完成供品仪式后，大门重新可达；若未完成两段家庭记忆，选择离开进入该结局。

“回煞”是家庭真相结局。玩家完成学校和医院谜题，理解父亲控制与母亲病情，但未拼出全家福或死亡线索不足，选择离开进入该结局。

“归路”和“永远在一起”属于完整真相。完整真相必须同时满足：学校谜题完成、医院谜题完成、全家福拼好、死亡线索至少 2 条并打开棺材。满足后进入车祸记忆；玩家必须先调查变形车门和护栏缺口，才能开放“沿公路离开”和“回老宅”两个选择。前者进入“归路”，后者进入记忆空间并由玩家再次确认后进入“永远在一起”。

低线索开棺不再自动把玩家送进团聚结局。棺材只补充真相并把玩家送回可行动状态，结局由重新开放的大门或完整真相的车祸记忆决定。

## 技术边界

`GameScene` 继续负责场景生命周期和系统装配，不再直接承载完整追逐状态机。新增或抽取的边界如下：

- `ObjectiveManager`：只根据进度状态生成目标文案并更新 DOM，不修改剧情状态。
- `ChaseManager`：负责生成、更新、结束和重试追逐者，向 `GameScene` 暴露开始、更新、躲藏成功和失败接口。
- 进度纯函数：放在 `StoryState` 或紧邻的独立模块中，负责谜题完成、完整真相判定、结局路由和当前目标推导。
- 谜题配置：使用独立数据文件保存学校与医院的题目、选项和正确顺序；通用浮层只读取配置并回调结果。
- `MapManager`：统一刷新静态物理体，按地图配置控制室内外雨、环境光和记忆变化。

交互数据流固定为：玩家调查或完成谜题 → 写入 `globalGameState` → 统一进度函数归一化并判定新阶段 → 目标提示刷新 → 地图物件或结局入口按状态开放。任何 UI 都不单独保存一份剧情真值。

新增源码仍通过 `tools/build_standalone_entry.mjs` 按依赖顺序打包到单文件 `index.html`，保持本地双击可玩。

## 稳定性修复

本轮必须覆盖以下已证实问题：

- 冲刺判定改为基于本帧输入方向，而不是清零后的旧速度；体力消耗和恢复使用 `delta`。
- 理智消耗使用 `delta`，60Hz 与高刷新率保持一致。
- 出生坐标使用空值判断，合法的 `0` 不再回退到默认坐标。
- 静态家具移动、缩放、设置碰撞箱或从状态恢复位置后统一同步物理体，重点覆盖父母衣柜和暗门。
- 移动端原生 DOM 监听器在场景关闭时完整解绑，重复换房不累积回调和定时器。
- 室内地图停用雨粒子；地图自定义环境光不再被 `GameScene` 统一覆盖。
- `SoundManager` 在进入 `GameScene` 时更新场景引用，空间音效不再长期引用 `TitleScene`。
- 追逐者从来源门生成；来源门不存在时，使用经过地图边界与墙体检查的可行走点。失败后恢复物理世界并从当前地图的 `playerStart` 重试，不回序章、不保留旧追逐实例和临时躲藏状态。

## 错误处理与兼容

`normalizeGameState` 为新增字段提供默认值，使没有新字段的旧运行状态仍可继续。非法地图 ID 继续回退到正常序章；谜题配置缺失时显示明确错误提示并允许退出浮层，不把玩家锁在页面上。

对话、文档、保险箱和谜题浮层必须互斥，关闭任何浮层后统一记录交互冷却，防止按键穿透导致同一物件立刻重开。移动端横竖屏都必须保留退出对白、谜题和躲藏的入口；竖屏可以保持 4:3 画面，但按钮和文字不能越出视口。

## 测试与验收

所有行为变更使用测试先行。最小自动化覆盖包括：

- 冲刺与体力在不同帧率下结果一致。
- 理智值在 60Hz 与高刷新率下按相同时间变化。
- `x = 0`、`y = 0` 出生坐标有效。
- 衣柜移动前后视觉位置与静态碰撞体一致。
- 场景多次重启后移动端监听器数量不增长。
- 学校和医院终点在谜题完成前不可用，错误答案不会污染完成状态。
- 四个结局的纯路由条件互斥且全部可达。
- 车祸最终选择在两项死亡现场调查完成前不可用。
- 追逐成功会停止追逐，追逐失败在当前章节重试。
- 室内无雨、地图环境光配置被保留。
- 18 张地图都声明非空 `purpose` 与 `rewards`，每张地图至少有一种独特产出。
- 地下室尺寸为 18×14，入口到照片缺角、照片缺角到出口均存在可行走路径，不含无产出的假岔路。
- 残缺全家福只有在地下室取得缺角后才能变为完整全家福。
- 3D 入口和运行依赖已从产品树及单文件入口中移除。

最后运行全部既有及新增验证、所有源码语法检查、单文件重建与契约检查，并使用浏览器完成桌面和移动尺寸烟测。人工流程至少覆盖开局、两段记忆、全家福、井边追逐、供品开棺以及“破茧”“回煞”和一条完整真相结局。

## 完成标准

玩家不再因为碰撞错位、错误重生或输入累积被卡住；主线始终有可理解的当前目标；18 张地图都有独特用途和可验证产出；地下室不再是空旷迷宫，而是高真相路线中目标明确、能顺利返回的地下禁闭室；学校、医院、照片和追逐至少形成三种不同的短玩法；四个结局条件能从游戏内反馈理解；2D 入口、源码和单文件构建保持一致；README、流程文档、路线图和 2026-07-11 技术日志与最终实现同步。

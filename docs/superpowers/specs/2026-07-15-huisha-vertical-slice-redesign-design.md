# 《回煞》实体解谜 Vertical Slice 激进重做设计

日期：2026-07-15

状态：老板已批准方向，等待书面规格复核

阶段：只设计 15–20 分钟验证切片，不扩写完整游戏

## 1. 决策摘要

《回煞》保留 Phaser 2D 技术底座、移动与交互聚焦、地图切换、跨图追逐预警和 BFS 寻路；停止继续扩展统一证据板、供品收集清单、随机惊吓、隐形理智值和按收集完成度分配结局。

第一阶段只重做正厅、厨房、孩子卧室三个主空间，目标不是做出一章内容，而是验证一条新的核心体验闭环：

> 发现房间中的异常规律 → 操作真实物件 → 观看矛盾记忆重演 → 修正判断 → 老宅永久改变 → 为自己的选择承担风险或失去保护。

只有切片通过五名陌生玩家盲测后，才允许把完整游戏重组为约 8–10 个高密度、可多次回访并持续变形的空间。当前 `codex/2d-rework` 分支和提交历史作为旧版归档，不删除、不重写历史；切片实施使用新的功能分支。

## 2. 已确认的根因

当前七个主要谜题都使用 `slots + tokens + answer`。这种结构只能表达固定配对，不能表达物件实验、空间因果、可观察错误、不可逆选择或世界状态变化。证据板又同时展示全部线索、槽位和选项，导致玩家离开环境后按文字相似度填表。

剧情同样被系统反复解释：交互标记、目标 HUD、线索文档、槽位名、失败提示、成功独白和 NarrativeDirector 会连续复述同一答案。厨房完成后已经直接说明空位属于主角，叙事阶段也会提前跳到“认知”，使棺材与车祸失去重新解释前文的力量。

现有地图虽然有 18 张，但多数只有一次访问价值；恐怖机制又分别使用随机抖屏、随机声音、理智值、纸人背视、数脚步和追逐，没有统一的世界规则。自动化主要验证字段、路由和状态可达性，已有试玩也使用过地图直达和前置状态装配，无法证明首次玩家真的经历了推理和情绪积累。

因此本轮不在旧谜题上继续换题面，而是更换承载谜题、叙事和恐怖的核心结构。

## 3. 产品目标与非目标

### 3.1 切片必须证明的体验

- 玩家会因为声音、光线、物件位置和房间结构主动调查，而不是追逐 HUD 清单或金色标记。
- 厨房座次答案必须由至少两条空间证据和一次可观察的矛盾重演共同支持，不能在任何文本中直接写出。
- 错误操作不弹“回答错误”，而是产生可用于下一次判断的世界反馈。
- 玩家通过观察学会一条恐怖规则，并在回访时主动使用该规则存活。
- 切片只制造“主角与失踪孩子有关”的怀疑，不确认主角死亡。
- 玩家在孩子卧室作出一个会立刻改变返程体验的选择，不存在无代价撤销。
- 切片结束后，玩家能记住具体场景、声音或动作，而不是只复述主角说出的结论。

### 3.2 本阶段明确不做

- 不重做全部 8–10 个最终空间，不恢复 18 张地图数量指标。
- 不实现学校、医院、后院、地下室、车祸和四结局完整内容。
- 不承诺兼容旧存档；旧版通过 Git 历史与分支保留。
- 不制作完整配音、多语言、成就、Steam 接口或最终美术资产。
- 不新增饥饿、弹药、背包格、程序化怪物或长期生存系统。
- 不用更多说明文字、提示按钮或谜题浮层弥补环境表达失败。

## 4. 切片空间与路线

切片只包含三个主空间，使用现有地图 ID 但重做内容与连接：

1. `room_main`：雨夜入宅、饭桌远景、三条感官引导和最终回访出口。
2. `room_kitchen`：实体饭桌座次、母亲纸人、第一次家规演示和侧门开启。
3. `room_bedroom_me`：纸飞机选择、儿童物件、镜像怀疑和返程代价。

正厅直接连接厨房；厨房正确完成后打开通向孩子卧室的侧门。孩子卧室同时保留一扇普通房门，经一段不可探索、没有线索的短走廊返回正厅。带走纸飞机时，玩家可沿侧门直接回厨房；留下纸飞机时，侧门会锁回，玩家必须从普通房门回正厅，再从正厅进入厨房。走廊不作为独立可探索地图，只承担这段短过渡和门外声源，不放线索或任务。

每个主空间必须具备至少两个持久状态：

- 正厅：初入冷暗 / 完成厨房后的饭桌回响 / 返程后的父影注意。
- 厨房：未摆桌 / 矛盾重演 / 正确饭局残影 / 根据纸飞机选择产生的安全或危险状态。
- 孩子卧室：封闭旧物 / 纸飞机被带走或留下 / 镜像与门锁变化。

## 5. 分钟级玩家体验

### 0–2 分钟：入宅

删除重复车祸说明。玩家从雨声中直接推开正厅大门，只听见厨房方向连续四次碗筷轻碰，画面中只能辨认三只日常碗和一只积灰冷碗。正厅不显示供品位置或地图清单；厨房滴水、微弱饭香和门缝暖光构成第一引导。

主角最多说一句事实性反应，不解释家庭历史。此时玩家应形成的唯一问题是：“为什么有人长期为第四个人留位？”

### 2–6 分钟：观察厨房

玩家可以检查并操作以下真实物件：

- 一把被钉死、能看住侧门与正厅门的椅子；
- 靠灶桌位上的药渍、烫痕和反复擦洗痕迹；
- 侧门门槛下的一块蓝边碎瓷；
- 水槽里的酒味缺口碗、药渍白瓷碗、蓝边纸飞机小碗；
- 已经放在第四个位置、积灰且盛着冷饭的供碗。

所有物件使用外观、位置、短音效和一行观察描述传递信息。关键物件不默认显示远距离金色呼吸点；只有玩家进入近距离并正对物件时才出现克制的交互轮廓。

母亲纸人第一次在玩家背身移动后更换锚点，不攻击、不弹说明。它的位置始终避开门口、桌位和关键物件。

### 6–10 分钟：实体座次谜题

玩家一次只能端起一只日常碗，并把它放到真实饭桌的三个可用座位。供碗不能被端走。桌面、碗和座位必须在游戏世界中可见，不打开 DOM 证据板。

每次形成完整但错误的三碗座次，厨房会在 4–6 秒内重演一段不一致的饭局残影：

- 酒碗不在钉死椅前：父亲残影起身，把侧门重新锁上；钉死椅发出拉扯声，提示这个位置属于控制出口的人。
- 药碗不在靠灶位：母亲残影端药时，碗在错误位置碎裂；残影结束后，药渍仍回到靠灶桌面。
- 蓝边小碗不在侧门位：门槛碎瓷发亮但无法与碗口缺口拼合，纸飞机残影撞在关着的门上。

反馈只指出世界矛盾，不显示错误槽位、正确答案或文字提示。玩家可以随时继续搬碗，不重置房间，不损失物件。

正确座次成立后，三个人物残影短暂入席，第四只冷碗仍无人触碰。冷碗开始积下从屋顶漏入的雨水，侧门自行打开。游戏只确认“有个孩子没有回来”，不使用“我”“明儿的忌日”“我已经死了”等揭底语言。

### 10–14 分钟：学习家规恐怖

侧门开启后第一次响起饭铃。第一次是无惩罚演示：三个人物残影同时停止，门外父亲脚步经过，母亲纸人保持静止。玩家即使移动也不会失败，但脚步会在门口停顿，让玩家注意到“饭铃与静止”有关。

第二次饭铃在玩家准备离开厨房时出现。若玩家保持静止约 2.5 秒，门缝阴影退去；若继续移动超过阈值，“家规执行者”会从玩家上一次经过的门进入检查。进入前必须依次出现敲门、脚步、门缝阴影三层预警，随后才开启 8–12 秒短追逐。玩家可停在桌下安全区或借侧门离开，失败只回到本次铃响前，不重做座次。

界面不得显示“别动”“它在听”或倒计时。规则通过残影和声音学习。删除切片中的随机抖屏、随机耳语、随机闪灯和理智值变化。

### 14–18 分钟：孩子卧室与代价

孩子卧室保留校服、折断纸飞机、未寄出的车票、漫画和过低的旧镜子。物件共同说明孩子准备离开，但不解释他后来是否死亡。

玩家必须对纸飞机作出一次不可即时撤销的操作：

- **带走纸飞机**：侧门保持开启，玩家获得更直接的返程路线；母亲纸人从厨房消失，不再指出桌下安全区，厨房也不再提供饭香与暖光保护。
- **留下纸飞机**：母亲纸人仍会在危险前指向安全区；侧门在玩家离开卧室后重新锁上，玩家必须经由正厅方向返回并面对更长的父影预警。

两个选项都必须同时包含收益和损失，文案不得使用“放下”“依赖”“正确选择”等道德标签。状态写入 `planeChoice`，切片内禁止读档撤销。

玩家最后一次看镜子时，镜中轮廓短暂降低到孩子身高，袖口与纸飞机照片中的孩子一致。主角不总结，镜像在玩家移动后恢复。

### 18–20 分钟：回访与结束钩子

玩家返回厨房。座次、纸人、门锁、暖光和父亲进入方向根据 `planeChoice` 与之前的饭铃表现改变。饭铃再次响起，玩家必须主动使用已经学会的规则完成返程。

切片终点设在正厅饭桌。冷碗水面短暂映出童年轮廓，随后雨滴打散倒影。画面结束在“这个失踪孩子可能与我有关”的怀疑上，不确认身份与死亡。

## 6. 系统设计与模块边界

### 6.1 `SliceState`

切片状态集中保存，避免把体验逻辑继续散入 `InteractionManager`：

- `bowlPlacements`：三只日常碗对应的真实桌位；
- `mealReplaySeen`：三类矛盾重演是否出现；
- `tableSolved`：座次是否成立；
- `houseRuleDemonstrated`：第一次饭铃演示是否结束；
- `fatherAttention`：`quiet / suspicious / checking / chasing`；
- `lastTraversedDoor`：父影进入方向；
- `planeChoice`：`null / take / leave`；
- `slicePhase`：`arrival / investigation / table / rule / bedroom / return / complete`。

状态转换由纯函数验证，不由 HUD 文本或物件销毁结果反推。

### 6.2 `KitchenTableController`

只负责拿起、放下、交换碗，维护三个桌位实体与碰撞/交互同步，并向记忆重演系统发送当前完整配置。它不显示答案、不发剧情奖励、不控制父影。

### 6.3 `MemoryReplayDirector`

接收完整错误配置，选择尚未看过且最能解释当前矛盾的一段作者重演。重演期间锁定碗操作但不锁死玩家移动；结束后恢复可操作状态。正确配置只触发一次成功饭局残影。

同一个错误不得反复播放长动画。已看过的矛盾再次出现时，只用 0.5–1 秒局部反馈提醒，避免试错坐牢。

### 6.4 `HouseRuleDirector`

统一管理饭铃、父亲注意度、预警、检查和短追逐。父影始终从 `lastTraversedDoor` 进入，复用现有跨图预警与 BFS 寻路。对话、记忆重演、物件搬运过场和场景切换期间暂停计时。

该系统不读取随机数决定核心惊吓，不写理智值，也不直接在 HUD 公布规则。

### 6.5 `SliceNarrativeDirector`

只管理当前阶段允许出现的短反应和禁止泄露的语义。切片阶段严禁出现以下确定性结论：

- 主角就是明儿；
- 七月十四是主角忌日；
- 主角已经死于雨夜；
- 棺材或供桌一直在等主角。

叙事反馈优先使用物件、残影、声音和房间变化；主角每个阶段最多一次一行反应，不替玩家总结谜底。

## 7. 交互、UI 与可访问性

- 继续使用最近距离焦点带，保证三个方向接近关键物件都能命中正确目标。
- 远距离不显示全图金色标记；关键物进入交互半径后才显示名称和动词。
- 拿碗时提示改为持有状态与可放置桌位轮廓，不弹背包或证据板。
- 正常游玩不显示主线清单、地图用途和“记忆总结”；暂停菜单可以提供一行无答案目标，例如“弄清这张饭桌少了谁”。
- 饭铃规则必须同时有声音、残影静止和门缝视觉反馈，确保无法依赖单一听觉通道的玩家仍可学习。
- 移动端使用同一套世界交互，不另做简化答案 UI；拖放可替换为“拿起—靠近桌位—放下”。
- 任何过场不得锁定关闭/跳过入口超过 6 秒；失败恢复不得超过 30 秒。

## 8. 恐怖与失败边界

父影不是随机刷新的怪物，而是家规的执行结果。核心规则只有一条：“饭铃响起时，家里的一切都必须停下。”纸人背视移动作为次级规律，但它既可能制造压力，也可能指向安全区。

父影注意度逐级变化：

1. `quiet`：只在门外经过；
2. `suspicious`：在玩家上次经过的门停顿并试门；
3. `checking`：进入房间检查最后移动位置；
4. `chasing`：发现玩家后进行一次短追逐。

玩家正确静止会降低一级注意度；连续违反才升级。任何失败只回到最近一次饭铃前，保留座次和纸飞机之前的持久选择。不得出现门口刷脸、无限追逐、地图重置、供品消耗或关键物件丢失。

## 9. 自动化与人工验收

### 9.1 自动化只证明机制契约

新增验证应覆盖：

- 碗只能占一个桌位，交换与恢复不会复制或丢失物件；
- 三类错误配置分别选择正确的矛盾重演；
- 正确座次不会依赖提示文本或 DOM 证据板；
- 饭铃期间静止、移动、对话、重演和切图的状态转换；
- 父影从最后经过的门预警并进入，追逐失败保留谜题进度；
- `planeChoice` 两条路线都可完成切片，且各自改变门、纸人和安全区；
- 切片源文本不包含提前揭底的确定性语句；
- 关键交互三个方向命中率与移动端控制契约；
- 构建、语法、控制台错误和旧版归档边界。

自动化通过不得被表述为“切片好玩”或“剧情有后劲”。

### 9.2 五名陌生玩家盲测是放行门槛

测试者不得提前阅读规格、代码、谜底或目标清单。必须从新档连续游玩，不使用地图直达、状态注入或开发者宝箱。记录完整录像、时间点、错误操作、口头判断和结束后访谈。

放行标准：

- 至少 4/5 玩家在入宅后 45 秒内产生第一次有目的调查；
- 至少 4/5 玩家在 8 分钟内无外部解释完成座次；
- 正确前错误完整配置的中位数不超过 2 次；
- 至少 4/5 玩家完成后能说出两条具体空间证据；
- 通过遍历全部组合解决的人数为 0；
- 至少 4/5 玩家第二次听到饭铃时主动预测或执行静止；
- 软锁、不可恢复状态、门口刷脸和无限追逐均为 0；
- 至少 4/5 玩家能说清“父亲控制孩子、母亲与孩子互相牵挂、孩子试图离开”；
- 切片结束时确信主角已经死亡的玩家不得超过 2/5；
- 至少 4/5 玩家能回忆一个具体意象或声音，而不是只复述文本；
- 纸飞机选择不得出现 5/5 一边倒；
- 无目标徘徊超过 60 秒的位置为 0；
- 至少 4/5 玩家主动表示愿意继续下一章节。

任何一项核心门槛失败，都不扩写完整游戏。先根据录像修订切片，再重新招募未接触过项目的玩家测试。

## 10. 旧系统迁移与归档

切片实施前从当前 `codex/2d-rework` HEAD 创建新功能分支。旧证据板、旧 18 图流程和四结局不在原分支上删除；切片分支只保证新入口与三个主空间可玩。

实施时按以下顺序隔离旧系统：

1. 新增切片专用状态与入口，不立即删除旧状态字段；
2. 让三个切片空间切换到实体谜题与新家规系统；
3. 从切片入口隐藏证据板、目标清单、随机恐怖和理智 UI；
4. 自动化与盲测通过后，再设计完整游戏迁移计划；
5. 完整重做获批前，不批量删除其余地图和结局源码。

这样可以随时从 Git 历史回到当前完整旧版，也能避免为了保留旧路线而污染新切片核心循环。

## 11. 被拒绝的替代方案

### 保守修补旧证据板

只改线索文本、减少提示和延后剧透，开发成本最低，但谜题仍发生在统一表单中，无法解决空间没有参与推理的问题。拒绝作为主方案。

### 在 18 张地图中只重做四个重点谜题

能改善局部，但会形成实体谜题与旧配对题并存的割裂版本，也要继续维护大量一次性房间和完成度结局。拒绝在验证新核心之前投入。

### 重写引擎或恢复 3D

当前问题来自内容架构，不来自 Phaser 或二维表现。重写引擎会扩大碰撞、交互、美术和性能风险，却不会自动产生更好的推理和人物弧。明确拒绝。

## 12. 完整游戏的后续边界

切片通过后，下一阶段才为完整游戏写独立规格与实施计划。届时预计把旧 18 张地图重组为约 8–10 个空间：公路/车祸合并，大门并入正厅，学校叠层进旧书房，医院叠层进父母卧室或药柜，密室并入墙腔，阁楼并入孩子卧室储物层，记忆空间回收到熟悉的厨房饭桌。

完整叙事按“谁没回来—为什么离开—我与孩子的关系—母亲的爱与依赖—雨夜死亡—谁还要替谁活”逐层揭示。结局由过程中的不可逆物件行为生成，而不是线索收集率。具体人物弧、四结局组合和全图状态机不在本切片规格中提前锁死，必须以盲测结果为输入另行设计。
